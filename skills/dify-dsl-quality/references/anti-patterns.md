# 反模式库

这些模式出现时，应优先重构而不是继续补字段。

## 1. 双入口混用

表现:

- `start` 与 `trigger-*` 同时存在

问题:

- 入口语义混乱
- 路径难以验证

## 2. 无终点

表现:

- 没有 `end`
- 没有 `answer`
- `knowledge-index` 不可达

问题:

- 流程不闭环

## 3. 死分支

表现:

- `if-else` 分支没有落点
- 分类节点类别无后继节点

问题:

- 节点永远不会被消费

## 4. 伪容器

表现:

- 有 `iteration` 或 `loop`，但内部没有真实处理节点

问题:

- 容器只增加复杂度

## 5. 变量悬空

表现:

- `value_selector` 指向不存在字段
- `assigner` 写入后从未被读取

问题:

- 流程表面完整，实际无效

## 6. LLM 滥用

表现:

- 简单模板拼接也上 LLM
- 简单条件判断也上 LLM

问题:

- 成本高
- 不稳定

## 7. 检索链路断裂

表现:

- `knowledge-retrieval` 结果没被消费
- `knowledge-index` 上游不是有效分块

问题:

- RAG 名义存在，链路不成立
