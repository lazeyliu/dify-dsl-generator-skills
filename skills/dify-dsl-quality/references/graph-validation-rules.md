# 图校验规则

用于生成后做结构校验。需要区分“源码硬校验”和“启发式审查”。

## 1. 源码硬校验

- 每个节点有唯一 `id`
- 每个节点有明确 `data.type`
- 每条边的 `source`、`target` 都存在
- 根节点类型与入口语义合法

## 2. 启发式审查

- 每个节点有唯一 `id`
- 每个节点有明确 `data.type`
- 必填字段完整

### 边级

- 每条边的 `source`、`target` 都存在
- `sourceHandle` 与节点类型匹配
- `targetHandle` 若存在，要与编辑器结构保持一致

### 路径级

- 入口能走到终点
- 没有明显死分支
- 没有孤立节点

### 容器级

- `iteration` / `loop` 都有内部起点
- 容器输出能被外部消费
- 容器不会无限扩散

### 变量级

- 选择器路径有效
- 变量作用域正确
- 写入后的变量能被读取

### 模式级

- `workflow` 有 `end`
- `advanced-chat` 有 `answer` 或解释
- `rag-pipeline` 有主链路尾部
- 触发模式没有无说明双入口
