**模式判断**
这是 `workflow` 模式，目标是对数组逐项处理并输出聚合结果。

**模板选择**
选择 `已验证稳定 Iteration` 骨架，核心路线是 `code -> iteration -> template-transform -> end`。

**节点清单**
1. `start_node`
2. `code_node`
3. `iteration_node`
4. `iteration_start`
5. `template_node`
6. `end_node`

**连边清单**
1. `start_node -> code_node`
2. `code_node -> iteration_node`
3. `iteration_start -> template_node`
4. `iteration_node -> end_node`

**字段检查表**
- `iterator_selector` 应指向数组来源
- `output_selector` 应指向容器内输出节点
- `start_node_id` 应指向 `iteration-start`
- `template_node` 应消费 `[iteration_node, item]`

**DSL 草稿**
```yaml
data:
  type: iteration
  iterator_selector: [code_node, result]
  output_selector: [template_node, output]
  start_node_id: iteration_start
```
