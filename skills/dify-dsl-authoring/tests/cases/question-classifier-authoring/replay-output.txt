**模式判断**
这是 `workflow` 模式，目标是让模型把输入路由到多个类别分支。

**模板选择**
选择 `条件分支模板` 的高质量版分支路线，因为它允许 `question-classifier -> end` 这种模型路由结构。

**节点清单**
1. `start_node`
2. `classifier_node`
3. `end_sales`
4. `end_support`

**连边清单**
1. `start_node -> classifier_node`
2. `classifier_node(sales) -> end_sales`
3. `classifier_node(support) -> end_support`

**字段检查表**
- `query_variable_selector` 应引用 `start_node.query`
- `classes[*].id` 应与边句柄一致
- `model` 需要显式声明

**DSL 草稿**
```yaml
data:
  type: question-classifier
  query_variable_selector: [start_node, query]
  classes:
    - id: sales
      name: 销售
    - id: support
      name: 客服
```
