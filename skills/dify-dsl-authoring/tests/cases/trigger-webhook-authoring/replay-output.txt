**模式判断**
这是 `workflow` 模式，但入口不是普通 `start`，而是 `trigger-webhook`。

**模板选择**
选择 `Webhook 触发` 路线，核心结构是 `trigger-webhook -> code -> end`。

**节点清单**
1. `trigger_node`
2. `code_node`
3. `end_node`

**连边清单**
1. `trigger_node -> code_node`
2. `code_node -> end_node`

**字段检查表**
- `method` 应显式声明
- `content_type` 应显式声明
- `body` 应定义 payload 结构
- 入口不应再并列普通 `start`

**DSL 草稿**
```yaml
data:
  type: trigger-webhook
  method: post
  content_type: application/json
```
