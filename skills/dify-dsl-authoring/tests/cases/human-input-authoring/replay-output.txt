**模式判断**
这是 `workflow` 模式，目标是在主链路中暂停并等待人工填写表单。

**模板选择**
选择 `审批/人工确认` 路线，核心结构是 `start -> human-input -> end`。

**节点清单**
1. `start_node`
2. `human_node`
3. `end_node`

**连边清单**
1. `start_node -> human_node`
2. `human_node(approve) -> end_node`

**字段检查表**
- `delivery_methods` 至少有一个启用项
- `user_actions` 至少有一个动作按钮
- `inputs[*].output_variable_name` 应唯一
- `timeout` 与 `timeout_unit` 应显式声明

**DSL 草稿**
```yaml
data:
  type: human-input
  delivery_methods:
    - type: webapp
      enabled: true
  user_actions:
    - id: approve
      title: 通过
```
