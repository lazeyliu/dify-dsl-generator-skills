**模式判断**
这是 `workflow` 模式，目标是从自然语言抽取结构化参数并输出到终点。

**模板选择**
选择 `基础 Workflow 模板`，在中间插入 `parameter-extractor` 作为结构化抽取节点。

**节点清单**
1. `start_node`
2. `extractor_node`
3. `end_node`

**连边清单**
1. `start_node -> extractor_node`
2. `extractor_node -> end_node`

**字段检查表**
- `reasoning_mode` 应显式填写
- `parameters[*]` 需要定义 `name / type / description / required`
- `end.outputs[*].value_selector` 应引用 `extractor_node`

**DSL 草稿**
```yaml
data:
  type: parameter-extractor
  query: [start_node, query]
  reasoning_mode: function_call
  parameters:
    - name: city
      type: string
      required: true
    - name: budget
      type: number
      required: true
```
