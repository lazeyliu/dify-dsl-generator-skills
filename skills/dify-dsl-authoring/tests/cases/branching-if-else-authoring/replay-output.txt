**模式判断**
这是 `workflow` 模式，目标是做显式规则分流，而不是 `advanced-chat` 或 `rag_pipeline`。

**模板选择**
选择 `条件分支模板`，最接近的变体是最小版分支骨架：`start -> if-else -> end`。

**节点清单**
1. `start_node`
2. `if_node`
3. `end_true`
4. `end_false`

**连边清单**
1. `start_node -> if_node`
2. `if_node(true) -> end_true`
3. `if_node(false) -> end_false`

**字段检查表**
- `if_node.data.type` 应为 `if-else`
- `cases[0].case_id` 应为 `true`
- `sourceHandle` 应与 `case_id / false` 分支匹配
- `end_*` 节点都应有 `route` 输出

**DSL 草稿**
```yaml
kind: app
app:
  mode: workflow
workflow:
  graph:
    nodes:
      - id: if_node
        data:
          type: if-else
          cases:
            - case_id: "true"
```
