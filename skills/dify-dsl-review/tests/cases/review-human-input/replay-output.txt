**模式判断**
- 这是 `kind: app` + `app.mode: workflow` 的人工审批 workflow，主链路是 `start -> human-input -> end`。

**节点清单**
- `start_node`
- `human_node`
- `end_node`

**连边清单**
- `start_node -> human_node`
- `human_node(approve) -> end_node`

**字段检查表**
- `delivery_methods` 至少有一个启用项
- `user_actions` 至少有 `approve/reject`
- `timeout` 与 `timeout_unit` 显式存在
- `form_content` 与输出字段 `comment` 一致

**风险分级**
- 阻塞项: 无
- 高风险项: `reject` 与 `__timeout` 恢复路径未显式展开
- 中风险项: 未做独立复核
- 优化项: 可补 email 投递配置

**最终结论**
- `需要人工确认`
