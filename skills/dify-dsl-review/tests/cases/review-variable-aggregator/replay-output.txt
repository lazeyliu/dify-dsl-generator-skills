**模式判断**
- 这是 `kind: app` + `app.mode: workflow` 的 fallback 聚合 workflow，主链路是 `code -> variable-aggregator -> end`。

**节点清单**
- `start_node`
- `code_primary`
- `code_backup`
- `aggregator_node`
- `end_node`

**连边清单**
- `start_node -> code_primary`
- `start_node -> code_backup`
- `code_primary -> aggregator_node`
- `code_backup -> aggregator_node`
- `aggregator_node -> end_node`

**字段检查表**
- `output_type` 为 `string`
- `variables` 按主值在前、fallback 在后
- `end` 输出引用 `aggregator_node.output`

**风险分级**
- 阻塞项: 无
- 高风险项: 无
- 中风险项: 未做独立复核；空值语义未做运行期回放
- 优化项: 可补 group_enabled 分组场景

**最终结论**
- `需要人工确认`
