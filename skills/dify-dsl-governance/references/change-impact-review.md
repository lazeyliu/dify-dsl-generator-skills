# 变更影响面审查

用于修改既有 DSL、替换模板变体、重排节点链路、合并能力块或准备发布前。

## 必查范围

- 节点增删改
- edge、selector、变量路径变化
- 输出约定变化
- fallback / 降级路径变化
- 外部依赖与工具调用变化
- 成本、时延、循环或 fan-out 变化

## 最小输出

| 范围 | 变更前 | 变更后 | 预期内 / 非预期 / 待确认 | 风险 | 说明 |
| --- | --- | --- | --- | --- | --- |
| 节点 / 变量 / selector / 约定 / 依赖 / 成本 |  |  |  |  |  |

## 审查规则

- 没有基线时，不要假装做了差异审查；明确写“无可比基线”。
- 影响关键路径的非预期变化，至少提升到高风险项。
- 输出约定、fallback、关键 selector 的变化若未说明原因，默认不是“优化项”。
- 仅当变更被清楚解释、风险被收敛、关键路径已覆盖时，才可视为“预期内变化”。

## 回归差异

如果存在旧 DSL、旧模板或旧样例回放结果，至少回答：

- 哪些变化是预期内的
- 哪些变化是非预期的
- 哪些变化需要人工确认

不要只写“有变更”。要指出具体节点、具体变量、具体输出或具体依赖。
