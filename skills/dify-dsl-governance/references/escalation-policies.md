# 升级条件

用于决定当前 DSL 什么时候应该澄清、回退、人工确认或停止执行。

## 默认升级条件

| 升级条件 | 触发条件 | 优先动作 |
| --- | --- | --- |
| 关键证据缺失 | 关键输入、关键标识、关键工具结果缺失 | 澄清 / 回退 |
| 关键证据冲突 | 多个证据来源得出互斥结论 | 人工确认 / 停止执行 |
| 高风险动作将被触发 | 会写状态、调用外部动作、生成关键结果链接或执行不可逆动作 | 人工确认 / 回退 |
| 上线前检查未通过 | 结构检查、输入输出约定检查、风险检查等失败或待定 | 停止执行 / 人工确认 |

## 规则

- 没有升级条件时，不要假定“默认继续执行”。
- 升级条件触发后的动作要写清楚，不能只写“提示用户”。
- 低置信度如果不会触发澄清、回退、拒答或人工确认，就不算真正定义了升级条件。
