# 覆盖率矩阵

用于修改既有 DSL、补做回归、准备发布、或判断“样例回放是否足够”。

## 最小覆盖维度

- 主路径
- fallback 路径
- 空结果路径
- 外部异常路径
- 低置信度路径
- 多轮状态路径
- 降级路径

如果某一类在当前 DSL 中不存在，可标记为“不适用”，但不要省略。

## 最小输出

| 路径类型 | 是否覆盖 | 样例 / 回放入口 | 结果 | 说明 |
| --- | --- | --- | --- | --- |
| 主路径 / fallback / 空结果 / 外部异常 / 低置信度 / 多轮状态 / 降级 | 已覆盖 / 未覆盖 / 不适用 |  | 通过 / 失败 / 未执行 |  |

## 规则

- 没有样例集、测试入口或回放输入时，不要写“已覆盖”；应写“未覆盖”，并说明原因。
- 主路径、fallback 路径、外部异常路径未覆盖时，至少提升到高风险项。
- 只有在适用路径覆盖率 `>= 95%`，且主路径、fallback 路径、外部异常路径均已覆盖时，才可把覆盖率判为“通过”。
- 如果做了“差异回放”，要写清旧版本与新版本在哪些路径上输出不同。
