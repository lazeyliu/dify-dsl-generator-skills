## 2. 校验摘要

- 先用 2 到 4 句自然语言总结整体结论，不要一上来只写状态码。

| 检查项 | 结论 | 这意味着什么 | 校验方式 / 依据 |
| --- | --- | --- | --- |
| YAML 语法 | 通过 / 未做 / 有问题 | 例如: DSL 文本可正常解析，当前未见 YAML 语法错误。 | 例如: 使用 YAML 解析器加载全文。 |
| 图结构 | 通过 / 未做 / 有问题 | 例如: 节点、连边与关键选择器未发现明显断裂。 | 例如: 按图规则检查节点 ID、source/target 与选择器引用。 |
| 代码片段 | 通过 / 未做 / 有问题 | 例如: `python3` 代码节点已通过静态语法检查，当前未见 `SyntaxError`。 | 例如: 对 `code` 节点执行静态编译或语法解析。 |
| 独立复核 | 已完成 / 部分完成 / 未做 | 例如: 已完成多方复核，未发现阻塞导入问题。 | 例如: subagent 角色结论或自检说明。 |

- 不要写成 `YAML_OK`、`STRUCTURE_OK`、`PY_CODE_OK` 这种裸标签。
- 如果必须保留机器可读标签，只能放在附注里，不能替代上面的自然语言摘要。
