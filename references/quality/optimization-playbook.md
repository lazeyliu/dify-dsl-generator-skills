# 编排优化手册

用于已有 DSL 的结构优化，而不是单纯补字段。

## 1. 减少无效节点

优先删除:

- 只做简单字符串拼接却使用 `llm` 的节点
- 只做变量透传却没有任何转换的中间节点
- 汇总后从未被消费的聚合节点

## 2. 降低链路深度

优先做法:

- 能在 `template-transform` 完成的，不要额外上 `code`
- 能在 `if-else` 完成的，不要额外上 `question-classifier`
- 能直接进 `end` 的，不要再绕一层中转

## 3. 提高可读性

优先做法:

- 分支节点后尽量保持节点职责单一
- 容器内部只保留与迭代或循环相关的节点
- 多分支汇总时优先使用显式聚合节点

## 4. 降低模型成本

优先替代:

- 文本拼接用 `template-transform`
- 硬条件判断用 `if-else`
- 简单数组整理用 `list-operator`

## 5. 提高稳定性

优先补强:

- `http-request.timeout`
- 默认值或失败兜底
- 人工确认节点
- 容器收敛条件
