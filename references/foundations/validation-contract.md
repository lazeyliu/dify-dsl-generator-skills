# 校验约定

只有 3 轮检查都完成，才可以把结果表述为“已精准校验”。

如果当前环境缺少解析器、实体模型、子代理能力，或者当前平台 / 会话策略要求额外授权，必须如实写出未完成项与原因，不要把缺失检查包装成“通过”。

## 第 1 轮：YAML 语法校验

目标：确认 DSL 文本本身可以被 YAML 解析。

至少说明：

- 是否完成
- 使用了什么解析方式
- 结果意味着什么

如果未做：

- 明确写“未做 YAML 语法校验”
- 不要使用 `YAML_OK` 之类的裸标签代替说明

## 第 2 轮：实体 / 模型级校验

目标：排除“YAML 能解析，但当前节点实体 / 模型会拒绝”的问题。

至少覆盖这些错误大类：

- 字段值域错误
- 字段存在性错误
- 字段类型错误
- 字段形状错误
- 字段组合关系错误
- 变量类型兼容性错误
- 运行时与内容一致性错误
- 历史兼容写法与当前实体写法失配

典型风险包括：

- 枚举值或操作符不在允许集合中
- 必填字段缺失或关键配置块为空
- `list` / `dict` / `bool` / `number` 被写成错误类型
- Python 运行时里混入其他语言语法

如果未做：

- 明确写“未做实体 / 模型级校验”
- 把最终结论至少降级到“需要人工确认”

## 第 3 轮：独立复核

目标：让独立视角检查字段遗漏、链路断裂、选择器悬空、历史兼容字段误用、提示词风险和最终结论是否站得住。

优先顺序：

1. 如果当前平台和会话策略允许自动使用子代理，优先 `3 个只读子代理 -> 1 个只读子代理 -> 0 个子代理` 退化。
2. 如果当前平台要求用户显式授权后才能使用子代理，先说明目的并取得授权，再进入上一条退化顺序。
3. 如果平台不支持或当前任务不允许，退化为完整自检，并明确写“未做独立复核”。

复核细节、角色提示词和模式选择见 [subagent-review.md](../quality/subagent-review.md)。

## 分级审查

3 轮检查之外，还要补 1 层分级审查，至少输出：

- 阻塞项
- 高风险项
- 中风险项
- 优化项

涉及提示词或模型节点时，至少审查：

- 精炼性与可执行性
- 指令冲突
- 变量准确性
- 业务合理性
- 结果准确性
- 多语言一致性
- 失败兜底与恢复
- 安全与注入

涉及复杂状态、工具链或多轮对话时，再补：

- 数据约定
- 状态污染
- 前置条件
- 决策覆盖
- 证据来源
- 多轮一致性
- 时效性
- 可观测性
- 成本时延
- 编排职责边界
- 副作用与幂等性
- 资源边界与终止性
- 回退路径约定一致性
- 敏感信息最小暴露
- 依赖与版本假设

更细的分级口径见 [graded-review-model.md](../quality/graded-review-model.md)。

## 结论判定规则

- `可直接导入`：3 轮检查已完成，且没有阻塞项，没有关键门禁失败。
- `需要人工确认`：没有阻塞项，但至少 1 轮检查未完成，或仍有高风险项 / 关键假设 / 关键门禁待确认。
- `明显不完整`：存在阻塞项，或者主链路、关键字段、关键约定明显缺失。

如果缺少基线、样例集、回放入口或真实导入验证，不要把这类证据空洞写成“通过”；按 [evaluation-gates.md](../governance/evaluation-gates.md) 与 [coverage-matrix.md](../governance/coverage-matrix.md) 如实标记为“待定”“未执行”或“无可比基线”。
