# 发布门禁

用于准备宣称“可直接导入”“已精准校验”或进入交付前。

## 默认门禁

| 门禁 | 默认要求 | 状态 |
| --- | --- | --- |
| 结构门 | YAML 语法、实体校验、图结构闭环完成 | 通过 / 失败 / 待定 / 豁免 |
| 契约门 | 关键输入输出契约、回退契约、依赖假设清楚 | 通过 / 失败 / 待定 / 豁免 |
| 风险门 | 阻塞项为 0，未关闭高风险项为 0 或已获人工确认 | 通过 / 失败 / 待定 / 豁免 |
| 回归门 | 若存在基线或样例集，适用路径覆盖率 `>= 95%`，且主路径、fallback 路径、外部异常路径不得缺测 | 通过 / 失败 / 待定 / 豁免 |
| 观测门 | 关键决策具备最小观测字段，不泄露敏感信息 | 通过 / 失败 / 待定 / 豁免 |
| 复核一致性门 | 多方复核冲突已归并；未归并项进入待确认 | 通过 / 失败 / 待定 / 豁免 |

## 使用规则

- 没有样例集或回放入口时，回归门默认是“待定”，不是“通过”。
- 只有在豁免理由清楚时，才允许使用“豁免”。
- “可直接导入”不应建立在结构门失败或风险门失败之上。
- “已精准校验”至少要求结构门、契约门、风险门和复核一致性门通过；回归门若存在入口，也应通过。
- 若定义策略档位但未显式指定，默认按 `strict` 解释。

## 结论映射

- `可直接导入`
  结构门、契约门、风险门通过；复核一致性门通过；其余门禁不存在失败项。
- `需要人工确认`
  没有阻塞失败，但存在待定门禁、豁免门禁或需业务确认项。
- `明显不完整`
  任一关键门禁失败，或关键门禁根本未做。
