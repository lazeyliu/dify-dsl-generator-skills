# 编排调优手册

用于在不大改结构的前提下调优表现。

## 1. 成本调优

关注点:

- LLM 节点数量
- 检索节点数量
- 工具调用频次

常见手段:

- 合并重复 LLM
- 把简单格式化逻辑改成模板节点
- 降低不必要的检索和工具调用

## 2. 延迟调优

关注点:

- 串行链路过长
- 外部请求过多
- 容器节点并发设置

常见手段:

- 提前过滤
- 减少串行外部依赖
- 合理使用 `iteration.is_parallel`

## 3. 质量调优

关注点:

- prompt 不稳定
- 检索结果不准
- 分支条件误判

常见手段:

- 调 prompt 模板
- 调检索参数
- 增加显式分支前置判断

## 4. 稳定性调优

关注点:

- 超时
- 重试
- 人工兜底
- 空值传播

常见手段:

- 增加 timeout
- 增加 retry
- 增加 human-input
- 给 end 输出设置更稳的来源
