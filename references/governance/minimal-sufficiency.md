# 是否已经够用

用于判断当前 DSL 是过重、合理还是过轻。

## 判断目标

- 结构过重
  节点过碎、能力重复、同一逻辑多处实现、成本明显偏高。
- 结构合理
  复杂度与目标匹配，职责清楚，关键风险有护栏。
- 结构过轻
  关键链路缺少护栏、缺少约定层、缺少必要 fallback 或缺少观测字段。

## 观察点

- 节点总数
- LLM 节点数
- 外部调用数
- fallback 数
- 归一化 / 适配层数量
- 循环 / 聚合复杂度
- 是否存在重复实现
- 是否存在单节点承担过多职责

## 最小输出

- 判断:
  结构过重 / 结构合理 / 结构过轻
- 主要依据:
- 可删减环节:
- 缺失环节:

## 规则

- 不要只按“节点越少越好”判断。关键看职责边界和风险控制是否匹配。
- 如果一个节点同时做抽取、决策、渲染，通常不是“精简”，而是职责过宽。
- 如果同一能力在多个地方重复实现，优先视为过重。
