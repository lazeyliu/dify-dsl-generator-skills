# 能力块契约

用于把 DSL 拆成可复用能力块、场景包或模板分层时。

## 每个能力块最小声明

| 字段 | 说明 |
| --- | --- |
| `name` | 稳定名称，优先职责导向 |
| `purpose` | 只做一件事 |
| `inputs` | 必需输入 |
| `outputs` | 稳定输出 |
| `preconditions` | 前置条件 |
| `side_effects` | 副作用 |
| `failure_semantics` | 失败语义 |
| `fallback` | 允许的降级方式 |
| `cost_class` | 成本等级 |
| `determinism` | 决定性程度 |
| `observability_fields` | 最小观测字段 |

## 边界规则

- 能力块优先按职责命名，不按具体业务对象命名。
- 策略层决定“何时用哪个能力块”，能力块本身不要偷带业务策略。
- 文案层负责表达，能力块不要承担最终渲染。
- 适配层负责外部差异归一化，能力块不要直接吸收所有脏结构。
- 如果一个能力块既做抽取、又做决策、又做渲染，默认视为职责边界过宽。

## 场景包

场景包可以复用多个能力块，但至少要声明：

- 入口条件
- 所需能力块
- 输出形态
- 失败降级路径

不要在场景包里复制能力块内部逻辑；优先组合，不要重复实现。

## 默认输出形态

- 面向用户阅读的默认输出形态
  `纯文本`
- 面向 API 或 DSL 流程消费的默认输出形态
  `JSON`
- 可选输出形态
  `Markdown`、`HTML`

如果没有明确需求，不要默认扩展到多语言并行输出或多形态同时输出。
